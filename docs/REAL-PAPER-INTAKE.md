# Real Paper Intake

Use this when confirmed original Track 1 papers are available.

## Why This Exists

`fixtures-real/` currently contains realistic local surrogate Track 1-style papers. They are useful for testing extraction mechanics, but they are not confirmed original Ralphthon Track 1 submissions.

Public search found event descriptions and submission mechanics, but not original Track 1 paper artifacts. When actual papers are available locally, replace the surrogate set with reviewed labels.

## Intake Command

Dry-run:

```bash
python3 scripts/prepare_real_paper_intake.py paper1.md paper2.md paper3.md --dry-run
```

Write pending fixtures:

```bash
python3 scripts/prepare_real_paper_intake.py paper1.md paper2.md paper3.md --output fixtures-real-pending
```

The script writes:

- `fixtures-real-pending/case-001/paper.md`
- `fixtures-real-pending/case-001/labels.draft.json`

`labels.draft.json` is not gold truth. It is generated by the current Review Agent and must be reviewed by a human before renaming to `labels.json`.

## Human Label Pass

For each case:

1. Read `paper.md`.
2. Open `labels.draft.json`.
3. Correct:
   - `expected_claim_count`
   - each claim's `expected_status`
   - each claim's `expected_evidence_refs`
   - limitation `source_id` and `text_contains`
   - `selected_next_experiment.target_claim`
4. Rename reviewed `labels.draft.json` to `labels.json`.

## Evaluation

After all pending cases have reviewed labels:

```bash
python3 scripts/evaluate_real_review_agent.py --fixtures fixtures-real-pending --run-id eval-real-confirmed-001
python3 scripts/evaluate_review_see_through.py --fixtures fixtures-unseen-3 --run-id eval-009 --archive
python3 scripts/summarize_runs.py --output runs/run_summary.md
```

If confirmed papers replace the surrogate set, copy the reviewed cases into `fixtures-real/` and rerun:

```bash
python3 scripts/evaluate_real_review_agent.py --fixtures fixtures-real --run-id eval-real-001
```
